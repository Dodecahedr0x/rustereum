# rustereum

An **experimental Rust DSL for EVM smart contracts.** You write contracts as
ordinary Rust structs and `impl` blocks annotated with `#[contract]`; rustereum
lowers them to Solidity, compiles that with `solc` (via
[`foundry-compilers`](https://crates.io/crates/foundry-compilers)) to EVM
bytecode, and lets you **inherit real OpenZeppelin contracts** through Rust trait
impls. Contracts are tested end-to-end in [`revm`](https://crates.io/crates/revm)
with a typed client generated by the macro.

> Status: a learning/experimental project, not audited or production-ready. The
> supported language is a small, deliberately-minimal subset (see
> [Language reference](#language-reference) and [Limitations](#limitations)).

```rust
use rustereum::prelude::*;

#[contract]
pub struct Counter {
    count: u256,
}

#[contract]
impl Counter {
    pub fn increment(&mut self) {
        self.count += 1;
    }

    pub fn get(&self) -> u256 {
        self.count
    }
}
```

## How it works

```
#[contract]  ──►  IR  ──►  Solidity  ──►  solc (foundry-compilers)  ──►  bytecode + ABI
   (proc macro)   (traits)  (lower)         │
                                            └─►  solc --ir  ──►  Yul (for inspection)
```

- The `#[contract]` proc macro lowers your struct/impl into a small **IR**
  (exposed as generated trait impls) and re-emits the original items so they
  still type-check as native Rust.
- An ordinary function lowers the IR to **Solidity** source.
- `solc` compiles the Solidity to **bytecode + ABI**. Because inheritance,
  dispatch, ABI encoding, and reverts are all handled by `solc`, real
  OpenZeppelin contracts "just work" as base classes.
- Every compile writes inspectable artifacts to `target/rustereum/`:
  `<Name>.sol` (generated Solidity), `<Name>.yul` (the Yul IR `solc` emits), and
  `<Name>.json` (bytecode + ABI).

## Testing contracts

The macro also generates a **typed client** and helpers, so tests drive the
contract in an in-process EVM without hand-writing selectors or ABI encoding:

```rust
use rustereum::testing::InMemoryDB;   // revm's in-memory DB, re-exported
use rustereum::vm::{Address, U256};

#[test]
fn counter_end_to_end() {
    let art = Counter::compile().expect("compile");   // Rust → Solidity → bytecode
    let mut evm = InMemoryDB::default();

    let counter = Counter::deploy(&mut evm, &art);     // typed deploy → CounterClient
    assert_eq!(counter.get(&mut evm), U256::from(0));  // view: decoded return
    counter.increment(&mut evm, Address::from([0x11; 20])).unwrap();  // state-changing: caller + Result
    assert_eq!(counter.get(&mut evm), U256::from(1));
}
```

- `Counter::compile()` assembles + compiles (the project root defaults to the
  crate's own directory); `Counter::compile_with(&opts)` overrides it.
- `Counter::deploy(&mut evm, &artifact[, ctor_args…])` returns a `CounterClient`.
- View methods (`&self`) take just the evm and return the decoded value;
  state-changing methods take an explicit `caller` and return `Result<_, Revert>`
  (so a rejected access-controlled call is `Err`).
- `revm` is optional, behind the `testing` feature; the generated client itself
  compiles without it.

## Inheriting OpenZeppelin

Inheritance is a **Rust trait impl**. Each imported Solidity contract has a
generated binding trait (created by `rustereum add`); implementing it declares
the base. `#[constructor(Parent(args))]` forwards constructor arguments to the
base (identifiers reference your own params; string/number literals are baked
verbatim), and `#[modifier(name)]` applies an inherited modifier.

```rust
use crate::bindings::Ownable;
use rustereum::prelude::*;

#[contract]
pub struct Counter { count: u256 }

#[contract]
impl Ownable for Counter {}                       // inherit OZ Ownable

#[contract]
impl Counter {
    #[constructor(Ownable(initial_owner))]        // → constructor(address initialOwner) Ownable(initialOwner)
    pub fn new(initial_owner: Address) {}

    #[modifier(only_owner)]                        // → the inherited `onlyOwner` modifier
    pub fn increment(&mut self) { self.count += 1; }

    pub fn get(&self) -> u256 { self.count }
}
```

See [`examples/openzeppelin/`](examples/openzeppelin/) for a gallery covering
`Ownable`, `Pausable`, `ReentrancyGuard`, `AccessControl`, `ERC20`, `ERC721`,
`ERC1155`, `ERC165`, and an `ERC1967Proxy` — and a note on what isn't reachable
yet (governance, interfaces).

## The CLI

`rustereum` scaffolds projects and manages Solidity dependencies. Imported
sources are **not committed** — they're declared in `rustereum.toml` and fetched
into a git-ignored `lib/`.

```console
$ rustereum new my-project        # scaffold Cargo.toml + rustereum.toml + src/
$ cd my-project
$ rustereum add OpenZeppelin/openzeppelin-contracts --ref v5.1.0
$ rustereum fetch                 # clone deps into lib/, regenerate remappings.txt + bindings
```

- `new <name>` — scaffold a project (`rustereum.toml` sits beside `Cargo.toml`).
- `add <owner/repo> [--ref <tag>]` — `git clone` a dependency, record it in
  `rustereum.toml`, write the remapping, and generate Rust trait bindings.
- `fetch` — reproduce a project's deps from `rustereum.toml` (what CI runs).

See [`crates/rustereum-cli/README.md`](crates/rustereum-cli/README.md).

## Language reference

The supported subset (v1):

| Category | Supported |
|---|---|
| Types (fields & params) | `u256`, `Address`, `bool` |
| Storage | struct fields (`self.field`) |
| Expressions | `self.field`, integer literals, `+` (`+=`, `=`) |
| Functions | `&self` (view) / `&mut self` (state-changing), typed params & return |
| Constructor | `#[constructor(Parent(args…))]` — args are param idents or string/int literals |
| Modifiers | `#[modifier(snake_case)]` (camelCased to match the inherited Solidity name) |
| Inheritance | `impl Parent for Contract {}` (single parent) |

Identifiers are written `snake_case` in Rust and converted to `camelCase` in the
generated Solidity (so ABI selectors are idiomatic).

## Limitations

Not yet supported (each maps to a concrete future feature):

- Only `u256`/`Address`/`bool` — no `string`/`bytes`/arrays as **parameter**
  types (string/number **literals** work as base-constructor args, which is why
  ERC20/721/1155 inherit fine).
- No arithmetic beyond `+`; no loops, locals, or conditionals in bodies.
- No **function overrides** with real bodies → contracts requiring overrides
  (e.g. `Governor`) can't be made concrete.
- No **interface implementation** or **external contract calls** → interfaces
  (`IERC20`) and cross-contract interop aren't expressible.
- No calling inherited `internal` functions with arguments (e.g. `_mint(...)`),
  so token examples prove inheritance + ABI, not minting.
- Single inheritance parent per contract.

## Development

```console
$ cargo build --workspace
$ cargo nextest run --workspace --features rustereum/testing   # revm end-to-end tests
$ cargo test -p rustereum --test ui                            # macro compile-fail diagnostics
```

The OpenZeppelin examples fetch their Solidity deps first:

```console
$ cargo build -p rustereum-cli
$ (cd examples/openzeppelin/ownable && cargo run -p rustereum-cli -- fetch)
```

CI ([`.github/workflows/ci.yml`](.github/workflows/ci.yml)) runs fmt, clippy,
fetches each example's deps, and runs the full test suite on stable Rust. The
first `solc` invocation downloads a pinned `solc` via `svm`.
